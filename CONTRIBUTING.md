# Contributing to Americano

Thank you for your interest in contributing to **Americano** — the cross-platform keep-awake CLI for NautilOS!

## How to Contribute

1. **Fork** the repository and create a feature branch for your changes.
2. **Read and agree** to our [Individual Contributor License Agreement (CLA)](CLA.md) before submitting any pull request.
3. Make your changes, add tests where appropriate, and ensure the code builds cleanly (`cargo build` and `cargo test`).
4. Submit a **Pull Request** to the `main` branch.

All external contributions require signing the CLA. The CLA ensures that contributors grant the necessary copyright and patent licenses while protecting the maintainer's pre-existing and unrelated patents.

## CLA Signing

We use a CLA process so that:
- You retain ownership of your copyright.
- You grant a license for your contributions to be used in the Project.
- The project remains legally clear for users and the NautilOS ecosystem.

Maintainer contributions (by dptrebas) are made under the same terms without needing to re-sign for every change.

## Development Setup

- This project is written in Rust.
- Install Rust via [rustup](https://rustup.rs/).
- Build and run: `cargo build` and `cargo run`

## Questions or Issues?

- Open an issue on GitHub.
- We welcome bug reports, feature suggestions, and documentation improvements.

By contributing, you agree to the terms in [CLA.md](CLA.md) and the [Apache-2.0 License](LICENSE).
